Что такое HTML?

HTML (HyperText Markup Language — «язык гипертекстовой разметки») — это стандартизированный язык для создания структуры и разметки веб‑страниц.

Кратко о главном:
Назначение: задаёт базовую структуру сайта — где будут заголовки, абзацы, изображения, ссылки и другие элементы.
Не является языком программирования: не выполняет вычислений или логики, а лишь «размечает» контент для браузера.
Основа веба: без HTML не существует ни одного сайта — это «скелет», на который «наращивают» дизайн (CSS) и интерактивность (JavaScript).

Как работает:
Состоит из тегов (например, <h1>, <p>, <img>), заключённых в угловые скобки.
Теги обычно парные: открывающий (<tag>) и закрывающий (</tag>).
Браузер читает HTML и отображает содержимое согласно разметке.

Структура документа

Основные элементы структуры HTML-документа

<!DOCTYPE html>
 Объявление типа документа. Указывает браузеру, что это HTML5.
<html>
 Корневой элемент. Охватывает весь документ. Может содержать атрибут lang (язык страницы).
<head>
 Содержит метаданные: заголовок, стили, скрипты, мета‑теги. Не отображается в основной части страницы.
<body>
 Основная часть документа. Всё, что здесь находится, отображается в браузере.
<title>
 Задаёт заголовок страницы (отображается во вкладке браузера). (внутри <head>)
<meta>
 Метаданные: кодировка (charset), описание (description), настройки viewport и др. (внутри <head>)
<link>
 Подключение внешних ресурсов (CSS файлов, иконок). (внутри <head>)
<script>
 Встраивание или подключение JavaScript кода.


    Семантические теги: 
    •  <header>
    • «Шапка» страницы или раздела.
    • Обычно содержит логотип, название, поисковую строку, главное меню.
    • Может быть несколько на странице (для разных разделов).
    •  <nav>
    • Навигационное меню (ссылки для перехода по сайту).
    • Используется для основных навигационных блоков (верхнее меню, боковое меню).
    • Не следует помещать в <nav> второстепенные ссылки (например, в footer).
    •  <section>
    • Логический раздел контента с собственной темой.
    • Должен содержать заголовок (h1 - h6).
    • Примеры: блок «О компании», раздел «Услуги»
    •  <article>
    • Самостоятельный фрагмент контента, который можно перепубликовать отдельно.
    • Подходит для: статей, постов в блоге, новостей, комментариев.
    • Внутри может содержать header, footer, section.
    •  <aside>
    • Дополнительное содержимое, связанное с основным контентом.
    • Примеры: боковые виджеты, реклама, «об этом вкратце», ссылки на смежные материалы.
    • Может располагаться внутри article или как отдельный блок.
    •  <footer>
    • «Подвал» страницы или раздела.
    • Обычно содержит: копирайт, контакты, ссылки на политику конфиденциальности, социальные сети.
    • Может быть несколько на странице (например, у каждого article).

Теги. Атрибуты. Элементы.

Тег

Тег (от англ. tag — «метка») — основной строительный блок HTML документа.
 - парные — состоят из открывающего и закрывающего тега;
 - одиночные (самозакрывающиеся) — не требуют закрывающего тега.

Классификация тегов

    Важно! в современной вёрстке предпочитают логическое форматирование и CSS. Физические теги считаются устаревшими.

  • По типу форматирования:
    1. Логическое (семантическое) форматирование:
      • Подчёркивает смысловую роль элемента, а не его внешний вид.
      • Браузер сам решает, как отобразить (на основе стандартов и стилей).
      • Зачем? улучшает доступность и SEO, помогает скринридерам и поисковикам понимать контент.
      • Примеры:

        <strong> — важный текст (обычно жирный);
        <em> — акцент (обычно курсив);
        <blockquote> — цитата;
        <cite> — источник цитаты;
        <code> — фрагмент кода.

    2. Физическое (визуальное) форматирование:
      • Прямо задаёт внешний вид (шрифт, цвет, размер).
      • Устаревшие теги (в HTML5 их заменяют CSS‑стилями).
      • Зачем? быстрое визуальное выделение (но лучше использовать CSS).
      • Примеры:

        <b> — жирный текст;
        <i> — курсивный текст;
        <u> — подчёркнутый текст;
        <font> — размер/цвет шрифта.

  • По типу отображения: (блочные vs линейные)
    1. Блочные элементы (block):
      • Занимают всю доступную ширину контейнера.
      • Начинаются с новой строки.
      • Могут содержать другие блочные или линейные элементы.
      • CSS-свойства по умолчанию: display: block; width: 100%.
      • Примеры:

        <div> — универсальный блочный контейнер;
        <p> — абзац;
        <h1>— <h6> — заголовки;
        <ol>, <ul> — списки;
        <section>,<article>, <header>, <footer> — семантические блоки.

    2. Линейные (строчные) элементы (inline):
      • Занимают только необходимую ширину под содержимое.
      • Располагаются в строке с другим текстом/элементами.
      • Не могут содержать блочные элементы (только линейные).
      • CSS-свойства по умолчанию: display: inline;, width зависит от содержимого.
      • Примеры:

        <span> — жуниверсальный строчный контейнер;
        <a> — ссылка;
        <strong>, <em> — логическое выделение;
        <img> — изображение (по умолчанию inline);
        <input> — поле формы.

Разница:
 - Блочные элементы создают «блоки» с переносом строки, линейные — «встраиваются» в поток текста.
 - Для изменения типа отображения используют CSS: display: block / display: inline / display: inline-block.

Частые теги и их функция:

 <h1>— <h6> — заголовки (от самого важного к второстепенному).
 <p> — абзац текста,хранит в себе текст и является блочным тэгом.
 <a href="..."> — ссылка (переходит по URL из href).
 <img src="..." alt="..."> — изображение (src — путь, alt — описание).
 <ul> — маркированный список.
 <ol> — нумерованный список.
 <li> — элемент списка (внутри <ul>/<ol>).
 <div> — блочный контейнер для группировки элементов.
 <span> — строчный контейнер для стилизации части текста,хранит текст и является строчным тегом.
 <br> — перенос строки (одиночный тег).
 <hr> — горизонтальная линия‑разделитель (одиночный тег).
 <section> — логический раздел контента.
 <article> — самостоятельный контент (статья, пост).
 <aside> — дополнительный контент (боковая колонка).
 <footer> — подвал страницы/раздела.
 <strong> — важный текст (семантика + визуал).
 <em>— акцентированный текст (семантика + визуал).
 <blockquote> — цитата с отступами.
 <code> — фрагмент кода (моноширинный шрифт).
 <pre> — предварительно отформатированный текст (сохраняет пробелы/переносы).
 <sup> - верхняя сноска
 <sub> - нижняя сноска
 <small> - меньше шрифт БЕЗ жирности

Атрибуты

Атрибуты — это модификаторы тегов, задающие дополнительные параметры элемента. Они записываются внутри открывающего тега в формате имя="значение" и определяют:
 - поведение элемента;
 - внешний вид;
 - взаимодействие с пользователем;
 - связь с другими ресурсами.

# - обращение к атрибуту в CSS
. - обращение к классу

Классификация атрибутов:

  • Глобальные — работают с любым HTML элементом: (id, class, style, title);
    • id — уникальный идентификатор элемента (для CSS/JS).
    • class — класс элемента (для группировки и стилизации).
  • Для ссылок (тег <a> и <link>): (href, target, rel, download)
    <a> - anchor - якорь - ссылка которая прикрепляется к тексту между тегом А.
    • href — URL ссылки
    • target — где открывать ссылку (_blank(в новой вкладке), _self и др.)
  • Для изображений (тег <img>):
    • alt — альтернативный текст для изображения (доступность, SEO).
    • src — путь к файлу (изображение, видео, скрипт).
    • width
    • height
  • Для форм (теги <form>, <input> и др.);
  • Для мультимедиа теги <video>, <audio>: (src, autoplay, loop, muted)
  • Для таблиц (теги <table>, <tr>, <td>, <th>);
  • Для таблиц (ARIA);
  • Для метаданных (тег <meta>): (name, content, charset, http-equiv)
    • name - создаёт якорь (точку перехода) внутри страницы.
  • События (запускают JS код при действиях пользователя):
    (onclick, onmouseover, onchange, onload и др)
  • Специальные (для отдельных тегов).
